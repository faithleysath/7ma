## 2. 请求/鉴权规范（由代码推断）

### 2.1 公共请求头
所有 `$http` 请求会附加以下请求头：
- `Accept: application/vnd.ws.v1+json`
- `Client: Wechat_MiniAPP`
- `Phone-Model` / `Phone-Brand` / `Phone-System` / `Phone-System-Version`
- `App-Version`
- 登录后自动追加：`Authorization: Bearer <token>`
- 条件追加：`X-Trace-Id`、`U-User-Id`、`GenieLamp-H-session`

对应代码：`common/main.js:72-73`, `common/main.js:98-124`, `common/main.js:265-299`。

### 2.2 Token 与登录态
- token 存储键：`token`
- 过期时间键：`token_expire_time`（本地时间戳）
- 请求前若已过期，直接按 401 处理，不再发请求
- 401 会清理本地登录态并引导到 `/pages/auth/index`

对应代码：`common/main.js:107-113`, `common/main.js:117-120`, `common/main.js:426-435`。

### 2.3 统一响应结构（推断）
业务代码普遍按以下结构读取：
```json
{
  "status_code": 200,
  "message": "ok",
  "data": {}
}
```

### 2.4 典型状态码处理
- `401`：未登录/过期，清 token 并跳登录
- `40301`：触发人机验证，跳 `/pages/40301/captcha-popup`
- `429`：限流（弹窗逻辑特殊处理）
- HTTP `600/429`：跳转 `/pages/http600/index`

对应代码：`common/main.js:410-413`, `common/main.js:435-445`。
