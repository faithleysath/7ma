# 7ma 定位/地图/登录接口详版（反编译代码推断）

> 目标：面向“实时采集车辆时空数据”场景，仅整理与 **登录鉴权、车辆定位、地图地标/运营区** 强相关接口。  
> 来源：`/pages/**/*.js`、`/common/**/*.js` 反编译代码静态分析。  
> 注意：本文示例数据为推断样例，不代表服务端完整字段。

## 1. 基础网关与请求规范

### 1.1 网关

- 业务网关（`baseURL`）：`https://newmapi.7mate.cn/api/v1`
- 认证网关（`authBaseURL`）：`https://newmapi.7mate.cn/auth`
- WebSocket：`wss://wss.7mate.cn:9501`

代码证据：
- `common/vendor.js:582-585`

### 1.2 公共请求头

统一 `$http` 封装会附加：

- `Accept: application/vnd.ws.v1+json`
- `Client: Wechat_MiniAPP`
- `Phone-Model`
- `Phone-Brand`
- `Phone-System`
- `Phone-System-Version`
- `App-Version`
- 登录后自动附加：`Authorization: Bearer <token>`
- 条件附加：`X-Trace-Id`、`U-User-Id`、`GenieLamp-H-session`

代码证据：
- `common/main.js:72-74`
- `common/main.js:98-124`

### 1.3 Token 与错误码关键行为

- 本地 token：`token`
- 过期时间：`token_expire_time`（毫秒时间戳）
- 401：清 token 并跳转登录页
- 40301：触发验证码验证流（`/pages/40301/captcha-popup`）
- 429/600：跳转网络异常页

代码证据：
- `common/main.js:107-113`
- `common/main.js:426-445`
- `common/main.js:410-413`

## 2. 登录与鉴权接口（重点）

### 2.1 `POST /auth/wechat/login`

微信 `code` 换取登录上下文（包含 token / openid）。

- 完整 URL：`https://newmapi.7mate.cn/auth/wechat/login`
- 鉴权：否

请求参数（代码可见）：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `code` | string | 是 | `wx.login` 返回 code |
| `device_id` | string | 是 | 设备标识，代码中拼接机型与 openid |
| `force_new_account` | bool | 否 | 403001 分支后用于“创建账号” |
| `restore_confirm` | bool | 否 | 403001 分支后用于“恢复账号” |

响应字段（代码读取）：

| 字段 | 用途 |
|---|---|
| `status_code` | 判断成功/分支（`200`、`403001`） |
| `data.session_key` | 登录态中间字段 |
| `data.token` | 保存为 `token` |
| `data.expire_time` | 转毫秒后保存 `token_expire_time` |
| `data.openid` | 保存 openid |

代码证据：
- `pages/auth/index.js:114-119`
- `pages/auth/index.js:126-133`
- `pages/minibus/index.js:90-95`

```json
{
  "code": "0f3xWk000ABCDEF",
  "device_id": "iPhone_14_Pro_xxxopenid",
  "force_new_account": false,
  "restore_confirm": false
}
```

响应示例（推断）：

```json
{
  "status_code": 200,
```

关键响应字段（代码可见）：

- `generate` 返回：`data.type`, `data.tips`, `data.background_img`, `data.slider_img`, `data.slider_y`, `data.token`

## 7. 实施提醒（和接口稳定性直接相关）

- `40301` 验证码是硬风控分支，不处理会中断采集流程（见 `common/main.js:435-443`）。
- 建议将采集状态机拆成：`token可用检查 -> 数据抓取 -> 401/40301/429分支处理`。
- 新版首页部分能力已迁移至 WebSocket ack（如 `car_location`、`in_fences`），仅靠 HTTP 可能缺实时性。
- 本文档仅基于前端反编译证据，最终以实测响应为准。
